# 1. 现有主流评测方法总结（你要先知道“行业怎么做”）

当前评估 agent 能力，基本分 5 类 benchmark 思路：

## 1. Tool / Agent 行为型（Agentic）

代表：

* GAIA（多步工具 + web + 推理）
* BFCL（function calling）
* AgentBench ([arXiv][1])
* τ-bench（对话式工具任务）
* Mind2Web（网页操作轨迹）

核心特征：

* 看 **是否做对动作序列**
* 是否完成任务（success / fail）
* 是否调用正确工具 API

👉 常用指标：

* Task Success Rate
* Step Success Rate
* Trajectory Score

---

## 2. 软件工程长程任务（Long-horizon coding）

代表：

* SWE-bench ([维基百科][2])
* Multi-SWE-bench
* Aider Polyglot
* LongCLI-Bench ([arXiv][3])

核心：

* 一次任务 = 多文件 + 多步骤 + debug + patch

👉 指标：

* Pass@1 / Pass@k
* Patch correctness (unit test)
* Regression rate
* Step completion ratio

---

## 3. Web / GUI Agent（真实环境）

代表：

* WebArena
* OSWorld ([arXiv][4])
* WebVoyager

核心：

* 在真实 UI 里操作（点击/输入/导航）

👉 指标：

* Task completion
* Navigation efficiency (steps vs optimal)
* Error recovery ability

---

## 4. Multimodal reasoning

代表：

* MMMU / MMMU-Pro
* VideoMMMU
* VQA / DocBench

👉 指标：

* Accuracy
* Grounded reasoning score
* OCR correctness / grounding score

---

## 5. Long-context / long dependency

代表：

* LongBench
* InfiniteBench
* RULER
* Needle-in-haystack

👉 指标：

* retrieval accuracy
* dependency consistency
* long-range coherence

---

# 3. 评测维度设计（核心）

## 维度A：Agentic Tool Use（工具执行能力）

### 测什么

* 是否能正确拆解任务
* 是否能调用工具
* 是否能多步修正错误

### 任务类型

* API调用
* 搜索 + 汇总
* 文件操作模拟
* function calling

---

### 自动评分（关键）

#### Score A =

```
A = 0.4 * Task Success
  + 0.3 * Tool Correctness
  + 0.2 * Step Efficiency
  + 0.1 * Error Recovery
```

---

### 可量化指标

| 指标              | 计算方式                                   |
| --------------- | -------------------------------------- |
| Task Success    | binary / 0-1                           |
| Tool Accuracy   | correct_calls / total_calls            |
| Step Efficiency | optimal_steps / actual_steps           |
| Recovery        | successful_repair_cases / total_errors |

---

---

## 维度B：Multimodal Task Orchestration（多模态编排能力）

### 测什么

* 图 + 文 + 视频 + OCR
* 是否能“跨模态串联推理”
* 是否能生成中间结构

---

### 任务设计参考 MMMU / VideoMMMU

---

### 自动评分

```
B = 0.5 * Answer Accuracy
  + 0.3 * Grounding Score
  + 0.2 * Cross-modal Consistency
```

---

### Grounding Score（关键创新）

可以自动算：

* 图像 → 文本一致性（CLIP similarity / embedding match）
* OCR match rate
* bounding box overlap（如果有标注）

---

---

## 维度C：Long-horizon Task Execution（长程任务能力）

### 来源参考：

* SWE-bench
* LongCLI-Bench ([arXiv][3])
* DeepPlanning ([Qwen][5])

---

### 测什么

* 10~50 step任务
* 中间状态会变
* 有失败恢复机制

---

### 自动评分核心设计

#### Score C：

```
C = 0.5 * Final Success
  + 0.3 * Intermediate Milestone Completion
  + 0.2 * Trajectory Stability
```

---

### 关键创新（你这个体系重点）

引入：

### 🧠 Milestone Graph Score（里程碑图评分）

把任务拆成 DAG：

```
Step1 → Step2 → Step3 → Step4 → Final
```

评分：

```
M = completed_nodes / total_nodes
```

---

### Trajectory Stability（轨迹稳定性）

衡量：

* 是否走回头路
* 是否反复错误
* 是否“漂移”

```
S = 1 - (backtrack_steps / total_steps)
```

---

---

## 维度D：Reasoning + Planning Quality（规划能力）

参考 GAIA + AgentBench

---

### 测什么

* 是否先规划再执行
* 是否能分解任务
* 是否 hallucinate tool

---

### 自动评分

```
D = 0.4 * Plan Quality
  + 0.3 * Plan-Execution Consistency
  + 0.3 * Hallucination Penalty
```

---

### Plan Quality（可自动评分方法）

用 LLM-as-judge + rubric：

* 是否分 step
* 是否完整覆盖目标
* 是否可执行

---

# 4. 总体评分模型（最终你要的量化结果）

## 🔥 FINAL SCORE（统一模型能力）

```
TOTAL =
  0.30 * A (Agentic Tool Use)
+ 0.25 * B (Multimodal Orchestration)
+ 0.25 * C (Long Horizon)
+ 0.20 * D (Reasoning & Planning)
```

# 5. 自动评测系统设计（工程实现）

## Step 1：统一 Task Schema

每个任务必须结构化：

```json
{
  "task_id": "...",
  "input": "...",
  "modalities": ["text","image","video"],
  "tools_allowed": [...],
  "gold_steps": [...],
  "final_answer": "...",
  "milestones": [...]
}
```

---

## Step 2：Trace Logging（关键）

记录：

* tool calls
* intermediate outputs
* reasoning steps
* final output

---

## Step 3：自动评估器

### (1) Rule-based evaluator

* exact match
* tool correctness
* step alignment

---

### (2) Embedding evaluator

* semantic similarity
* image-text alignment

---

### (3) LLM-as-judge（必备）

用于：

* plan quality
* reasoning coherence
* multimodal consistency
