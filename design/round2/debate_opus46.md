# 第 2 轮辩论 · 设计者-Opus-4.6（CORE-A）对 ARGUS-Eval 与 VERITAS-Bench 的交叉质疑、回应与修订

**作者：设计者-Opus-4.6** | **轮次：Round-2 Debate** | **日期：2026-06-25**

> 本文针对 GPT-5.5 的 **ARGUS-Eval**（6 维 D1–D6 + typed evidence graph + IRT 2PL）与 Opus-4.8 的 **VERITAS-Bench**（8 维 C1–C8 + MIRT + grounding by construction + 因果有效性门控）展开。结构：§0 三方共识与真正战场 → §1 质疑 ARGUS → §2 质疑 VERITAS → §3 一个同时命中两者的致命共性（小样本 IRT）→ §4 诚实回应对我的攻击 → §5 CORE-A v2 修订 → §6 统一方案建议（含 7 个必答表态）→ §7 不让步点 / 最大让步 / 三条最关键批评。

## 0. 三方共识与真正的战场

先把"已经吵完的"清出去，避免在辩论中浪费火力。三份设计已经**强收敛**于以下几点，应直接写进统一方案，不再争论：

| 已收敛的共识 | Opus-4.6 | GPT-5.5 | Opus-4.8 |
|---|---|---|---|
| CLIP/embedding 不能做 grounding 判定指标 | ✅(LSVF) | ✅(typed evidence) | ✅(construction) |
| 不以"单一总分"为 headline | ✅(Profile) | ✅(向量+稳定性) | ✅(θ画像) |
| 统计严谨性是发布前置（CI/显著性/多重校正） | ✅ | ✅ | ✅ |
| 反污染是架构级（程序化生成+私有held-out+canary） | ✅ | ✅ | ✅ |
| LLM-judge 是最后手段，需可靠性工程+人类校准 | ✅ | ✅ | ✅ |
| 对抗鲁棒性应是一级维度 | ✅(A) | ✅(D5) | ✅(C8) |
| 里程碑=带 verifier 的依赖 DAG，反对 `completed/total` | ✅ | ✅ | ✅ |
| 效率必须"成功条件化"、不能奖励"更快地失败" | 部分 | ✅ | ✅ |

**真正的战场在这 6 个分歧上**，本文火力集中于此：

1. **聚合机制**：MIRT（4.8）vs 混合效应模型（5.5 也用）vs 我的乘法聚合 —— 在 `N_models ≈ 5–15` 这个残酷现实下，谁是统计上诚实的？
2. **效率是否允许进 capability 分**：5.5 折入 ≤15%（且与其自己 §12.7 矛盾）vs 4.8 完全移出 vs 我（原来也折入，本轮让步）。
3. **grounding 的可验证性 vs 生态效度**：4.8 的"程序化合成（synthetic）"换来符号级 GT 但牺牲真实分布；我的 LSVF 用真实资产换 GT 噪声；5.5 的 evidence graph 引入标注成本与"本体序列化"陷阱。
4. **因果有效性门控怎么实现**：4.8 的"反事实空跑基线"在路径依赖环境里是否良定义。
5. **维度数**：5 vs 6 vs 8 —— 在有限区分度预算下，切得越细越好吗？
6. **pass@k vs pass^k 谁当 headline**。

## 1. 对 ARGUS-Eval（GPT-5.5）的尖锐质疑

### 1.1 §10.3 soft event matching 把它自己刚批判的"任意权重 + 相似度软分"又请了回来 —— 内部矛盾【严重程度：高】

GPT-5.5 在 §1.1/§1.2 严厉批判固定权重与相似度软指标，但在 §10.3 的 milestone soft matching 里写下：

```
match_score(r,o) = λ_type·1[type compatible] + λ_target·sim_target + λ_arg·sim_args
                 + λ_state·state_delta_match + λ_evidence·evidence_match
c_j = β·S_j + (1-β)·state_verifier_j
```

- **技术问题一（任意权重复活）**：`λ_type, λ_target, λ_arg, λ_state, λ_evidence` 这 5 个权重和 per-task 的 `β` 从哪来？GPT-5.5 只说"高风险副作用任务 β 应较低"，没有给出确定 λ/β 的可辩护程序。这正是它自己定义为"不严谨"的固定权重，只是从总分层挪到了里程碑层。
- **技术问题二（相似度软分复活）**：`sim_target`、`sim_args` 是相似度函数。如果用 embedding 实现，它就是 §1.3 自己痛斥的"度量相似而非正确"；如果用规则实现，则又退回 schema 校验（那 soft matching 就没必要存在）。两难。
- **可被 gaming**：`λ_arg·sim_args` 给"参数长得像"打分。模型可以输出"参数形似但语义错"的调用来骗取 `S_j` 的部分分，再靠 `β` 把状态 verifier 稀释掉。
- **建议修正**：把 soft matching 限制为**纯离散的类型/目标对齐**（`1[type]·1[target_id 命中]`），删除连续 `sim_*` 项；`c_j` 不做 `β` 线性混合，而是 **状态 verifier 一票否决**（状态没达成则 `c_j=0`，event matching 仅用于在多条合法路径间做归属，不贡献独立分数）。这恰好是 4.8 §5.1 与我 v2 的做法。

### 1.2 Evidence Graph（§10.4）把"会推理"和"会按我们的本体序列化"混为一谈 + GT 标注不可扩展【严重程度：中高】

GPT-5.5 要求模型输出 typed evidence graph，按 `F1_nodes + F1_relations + citation_consistency` 打分，gold 是 `E*`。

- **能力混淆**：一个模型可能做了完全正确的 grounded reasoning，但没有以 `{OCRToken, BBox, VideoSpan, supports, contradicts, located_at, ...}` 这套本体序列化输出，于是 `F1_relations` 偏低。这测的是"本体顺从度（ontology compliance）"而非"证据推理能力"。强模型若用自己的表述组织证据反而吃亏。
- **标注成本与覆盖面**：为一段 20 分钟仓库监控视频（§6 D3-T1）构造 gold evidence graph（含逐对象 track、bbox、temporal span、因果边）极其昂贵，而且每个**参数化变体**都要重新生成一份一致的 `E*`。GPT-5.5 标榜"template+generator+private params"可规模化，但 evidence graph 的 GT 在非合成资产上**无法**随参数廉价再生 —— 这与它的可扩展性主张冲突。
- **建议修正**：（a）不把 evidence graph 作为**模型必须产出的评分对象**，改为"从模型自由输出中用受约束抽取器抽证据 → 对符号 GT 判定"，并报告抽取器自身在 held-out 上的 F1（抽取器是受测仪器）；（b）evidence graph 的 GT 只在**程序化合成资产**上自动派生（与 4.8 的 construction 思路合流），真实资产上退化为关键节点（bbox/CER/数值）的稀疏验证。

### 1.3 §10.7 把效率折入 `Score_t`（≤15%）与 §12.7"主榜不做 cost-adjusted"自相矛盾【严重程度：中高】

- §10.7：`Score_t = A_t·[(1-η_t)+η_t·Eff_t]`，`η_t ≤ 0.15` —— 效率**进入了 per-task 能力分**。
- §12.7：明确写"主榜不应默认使用 cost-adjusted，否则把'便宜但弱'与'贵但强'混进能力测量"。
- **这是直接矛盾**：`Score_t` 是能力主分的构件，它已经被 `Eff_t` 调制了最多 15%。于是两个能力完全相同、只是 verbosity 不同的模型会得到不同的 `Score_t`，能力测量被污染——正是 §12.7 想避免的。
- **附加问题（C_ref 是隐藏的拍脑袋量）**：`Eff_t=exp(-λ·max(0, C_t/C_ref−1))` 依赖 `C_ref`（由"强基线/人类轨迹/难度"估计）。`C_ref` 宽则人人 `Eff≈1`（无区分），紧则惩罚合法探索。GPT-5.5 对 `C_ref` 与 `λ` 没有敏感性分析，等于把效率分的尺度交给一个未审计的超参。
- **建议修正**：把效率**彻底移出** `Score_t`（采纳其自己的 §12.7 与 4.8 §5.3 立场），只在能力–成本帕累托前沿上报；若要标量化，单列 `U=Success−λ·Cost` 且对 `λ` 做敏感性。

### 1.4 缺"因果有效性"门控，里程碑仍可被"环境免费满足"刷分【严重程度：中】

GPT-5.5 §10.2 有依赖门控 `ĉ_j = c_j·min_u max(ĉ_u, ρ)` 和回归惩罚 `min(c_ever, c_final)`，但**没有** 4.8 §5.1 的 `1[achieved-by-agent]`（因果归因）。

- **后果**：若某里程碑谓词在初始状态恰为真，或由环境动态（非 agent 动作）变为真，GPT-5.5 会照计分。例如 D4-T1 数据迁移里，某个"invariant 成立"可能是初始就成立的，模型什么都没做也拿分。
- **建议修正**：引入因果归因（见我 §5.R2 的"工具效应归属"实现，比 4.8 的全局空跑基线更稳健）。

### 1.5 D1–D6 仍有耦合，且未给正交性验证方法【严重程度：中】

GPT-5.5 批判 AGENIX 维度耦合，但 D1（Goal-State Agency）与 D5（Robust Tool Interaction）都以工具交互为核心，D2（Contingent Planning）与 D1 在"达成目标态需要规划"上重叠。它的发票任务（D1-T1）显然同时压 D2（规划）与 D6（不盲目自动化=校准）。

- **关键缺失**：GPT-5.5 **没有给出任何正交性的事后验证**（我有因子分析/相关矩阵，4.8 有 MIRT 载荷纯度）。它的 6 维是"声称正交"而非"验证正交"。
- **建议修正**：要么补上维度间相关矩阵 + 载荷纯度报告，要么承认 D1/D2/D5 是"任务表面"而非"能力轴"（这恰恰是 4.8 §3 的解耦主张，GPT-5.5 应当吸收）。

### 1.6 D6 校准维度的"可程序化判定"被高估【严重程度：中】

D6-T1（法律尽调）成功判定含"memo 中不夸大确定性"、"给出 calibrated uncertainty"。

- **问题**：这些是**质性**判断，要么落回 LLM-judge（GPT-5.5 想最小化），要么靠脆弱的关键词规则。真正的校准测量需要：模型输出数值置信 + 多任务上计算 ECE/Brier/可靠性曲线。单任务"是否恰当地 hedge"只测了"在这道被设计成该 hedge 的题上有没有 hedge"，模型可以**学会无脑 hedge** 通过所有此类里程碑，而全局校准极差。
- **建议修正**：把校准从"单任务里程碑"升级为"跨任务的概率校准统计量（ECE/Brier）+ 选择性预测（risk–coverage 曲线）"。这是三方共同的盲点（我 E、4.8 C7 同病），建议统一方案专门处理（见 §6.3）。

### 1.7 小结
ARGUS-Eval 工程完整度很高（trace schema、record/replay、混合效应模型都很好），但它的**内部一致性**有两处硬伤（§10.3 软匹配权重、§10.7↔§12.7 效率矛盾），且把 evidence graph 的成本与本体顺从问题低估了。

## 2. 对 VERITAS-Bench（Opus-4.8）的尖锐质疑

### 2.1 MIRT 是当前样本量下的"统计戏剧"——这是核心过度工程【严重程度：高】

4.8 把 8 维 MIRT 作为聚合与归因的**中枢**：`P(item i 通过|θ)=σ(a_iᵀθ − b_i)`，并用 Fisher 信息 `I_i(θ)=a_i a_iᵀ P(1−P)` 选题/做 ceiling。

- **致命问题——examinee 数太少**：IRT 的 item 参数（难度 `b_i`、判别 `a_i`）精度随**考生数**增长。教育测量里 2PL 稳定标定通常要 ≥500 名考生。这里"考生"= 被比较的模型 = 现实中 **4–20 个**。用 ≤20 个考生去联合估计**每题 8 维判别向量 + 难度**，是严重欠定的。
- **贝叶斯正则=自证**：4.8 在 Q2 用"载荷先验 + 主压 1–2 维 + 分层贝叶斯 + CI"辩护。但这恰恰**自认其罪**：当后验被先验主导时，`θ` 估计≈作者先验的重参数化，`a_i`"数据驱动判别"其实就是作者标注的载荷 `w_i`。于是 §5.7 的"数据驱动权重"和 §8.2 的"Fisher 信息选题"建立在**先验指定**的 `a_i` 上——**循环论证**。
- **边际价值≈0**：MIRT 相对"逐维均值 + cluster bootstrap CI"（4.8 §5.8 自己已有）的全部增量，正好是判别加权与难度标定——而这部分正是小考生数下最不可靠的。换言之，MIRT 增加了 MCMC 收敛、先验敏感、可识别性三重脆弱，却没换来可信的增量。
- **建议修正**：当前 `N_models` 下用**混合效应模型**（GPT-5.5 §5.8 已采用）作为主估计器：`logit P(success)=θ_M + u_template + u_instance`，模型为固定效应、模板/实例为随机效应。它给出"类能力"的模型对比 + CI，**不需要**估计 per-item 判别向量，是 5–15 考生下统计诚实的中间地带。MIRT 仅在未来考生数足够大（如把大量 checkpoint 当考生，且满足局部独立）时作为**二级诊断**，并强制报告先验敏感性。

> 注：把"多 checkpoint/多温度/多版本"灌进来凑考生数不能救场——它们高度相关，违反 IRT 的局部独立与总体假设，而且我们关心的是**特定已发布模型**的对比，不是一个总体。

### 2.2 "因果有效性"的反事实空跑基线在路径依赖环境里不良定义【严重程度：中高】

4.8 §5.1 用 `1[achieved-by-agent(v)]`，判据是"若不执行任何 agent 动作该谓词也为真则不计"。4.8 自己在 Q1 也担心这点。

- **技术问题**：在动态、路径依赖环境里，环境演化**依赖 agent 动作**，"空跑"轨迹与真实轨迹根本不可比。以 4.8 自己的 `tf2.config_migration` 为例：schema 在"迁移完第 7 个服务"时才切换 v2.1——空跑 agent 永远触发不了这个切换，所以"空跑世界"里 G1/G2 的谓词恒假。拿一个**永不进入相关状态**的反事实世界做基线，无法判定"该状态是不是 agent 造成的"，只能得到平凡结论。
- **更深问题**：正确的因果归因需要良定义的干预与可比反事实；单一"全空跑"是个糟糕的反事实（它改变了整条因果链，而非局部干预）。
- **建议修正（我 v2 采用、并优于 4.8）**：**工具效应归属（tool-effect attribution）**——每个 mock 工具声明它能改写的状态变量集合；里程碑谓词只在"其依赖的状态变量被某个 agent 工具调用的因果锥所改写"时才计入。这是局部、可程序化、与路径无关的因果判定，避免脆弱的全局空跑。

### 2.3 grounding by construction：生态效度损失，且合成资产可能**更**容易、压低顶端区分度【严重程度：中】

4.8 全押程序化合成（matplotlib 图、TTS 视频、合成 3D 场景）以换符号 GT。

- **生态效度**：真实多模态 agent 面对的是脏扫描件、自然光照、遮挡、手写。合成图表上满分不代表真实图表上能行。4.8 在 Q4 自认这点。
- **被低估的反向问题**：matplotlib 默认风格在训练语料里**极其常见**，合成图表可能比真实图表**更易读**，于是顶端模型一起饱和，反而**降低**高 θ 区的区分度——与 4.8 自己"最大化对顶级模型区分信息"的目标冲突。同时渲染管线本身成为混杂：擅长"matplotlib 风格读数"的模型虚高。
- **建议修正**：双轨——合成最小对（Winoground 式，符号 GT）专门做**顶端区分探针**；真实资产 + 结构化 verifier（我的 LSVF：检测器/OCR 引擎 + 容差）做**生态效度**，二者**分开报告**，不混入同一 grounding 标量。

### 2.4 轮换 held-out + 锚定 item 等值化是自我拆台【严重程度：中高】（本轮题面点名）

4.8 §6.2/§8.3 用"私有集定期轮换 + anchor item 做 IRT linking"解决纵向可比。

- **悖论一（锚点最易被污染）**：要做跨版本桥梁，anchor item 必须**长期不变**——于是它们因持久存在而**最大程度暴露**于训练污染。4.8 在 Q9 自己提出"锚定项会否被污染从而污染等值化链条"。这不是小瑕疵，是等值化链条的根基被蛀。
- **悖论二（linking 依赖不可识别的 IRT）**：IRT linking（Stocking-Lord/mean-mean）需要稳定的 item 参数——又回到 §2.1 的小样本不可识别问题。等值化继承了 MIRT 的病。
- **建议修正（我 v2 采用）**：**按模板/难度分布锚定，而非按字面 item 锚定**。桥梁集是"锚模板的全新种子同构实例"（同分布、新表面），用它的**难度分布**而非固定题面做等值化参照。这样既抗污染（每次新种子），又保持跨版本难度可比；配合 isomorph-gap 监控锚模板是否已被记忆。

### 2.5 必要性测试定义 `c*` 的可行性：组合爆炸 + 随机环境 + 贪心非最优【严重程度：中】

4.8 §5.3 用 oracle 解的"逐动作类消融"定义最小成本 `c*`。

- **贪心 ≠ 最小**：动作之间有交互，逐类移除（greedy）得到的不是最小必要集合（这是经典的子集选择 NP 问题）。
- **随机环境下 `c*` 是分布不是点**：4.8 的任务大量注入 flaky 工具/瞬时失败。"移除某动作后是否仍成功"本身是随机事件，于是 `c*` 是随机变量，`Regret=(c_model−c*)/c*` 的分母带噪。
- **oracle 不总存在**：开放任务（多路径、需创造性）未必有近最优 oracle solver。
- **建议修正**：`c*` 用"强基线分布的低分位数（如 P10 成功轨迹成本）"作为**稳健参考**，避免组合消融；在随机环境下对 `c*` 取多次重采样中位数并报告其 CI；无 oracle 的任务退回"强基线参考 + 帕累托"，不强行算 regret。

### 2.6 8 维在有限区分度预算下过度参数化【严重程度：中】

- **统计**：`N_models ≈ 5–15` + 有限任务预算下，切成 8 维 → 每维由更少 item 支撑 → 每维 CI 更宽。维度数有 bias–variance 权衡：越多维归因越细，但每维统计功效越低。8 维 MIRT 是三方中最过参数化的。
- **概念重叠**：C2（Tool Grounding）vs C6（Constraint Adherence）——调用禁用工具既是 C2 选择失败又是 C6 约束违反；C1（Planning）vs C3（State Tracking）在动态重规划中难分。8 维并不比 5–6 维更正交。
- **建议修正**：把 8 维**折叠为 5–6 个一级能力轴**（用于 headline 与归因），C2/C6、C1/C3 各自合并或设为"同一轴的两个 sub-probe"；保留 8 维仅作诊断子标签。

### 2.7 pass^k 当 headline：高方差估计 + 温度混杂【严重程度：中】

4.8 §5.8/Q6 倾向 pass^k（k 次全成）作为 agent 主指标。

- **估计问题**：pass^k 是对 per-run 成功率 `p` 的强非线性变换（`p^k`）。真值 `p=0.8, k=5` 时 pass^5=0.33；用 5 次运行去估 pass^5 方差极大，要可靠估计需 `n≫k` 次运行。
- **温度混杂**：temp=0 时 pass^k≈pass@1（平凡），temp>0 的"有创造性"模型被惩罚。于是 pass^k 测的是**部署旋钮（温度）**而非能力。
- **建议修正**：headline = **per-run 成功率（mean±CI）或混合效应 θ_M**；pass@k（上限/可达性）与 pass^k（可靠性/一致性）作为**互补列**，明确标注 k、要求 `n≫k`、并报告温度设定。

### 2.8 tripwire 把"致命违规"与"轻微越界"一锅烩零【严重程度：低中】
`Success=∏φ_p·∏1[tripwire 未触发]` 用硬乘积，一个轻微约束滑动（如多提交一张工单）就把整题 Success 清零。Success 作为二元结果这样做尚可，但缺**严重度分级**（critical vs minor）。建议：critical（泄密/不可逆副作用/honeypot）才清零并计入 ASR；minor 约束滑动在其所属里程碑内扣分，不全局清零（诊断信号不该被一次小错抹掉）。

## 3. 一个同时命中 GPT-5.5 与 Opus-4.8 的致命共性：用 ~5–15 个"考生"做 IRT

这一条值得单列，因为它**同时**击中两份设计的统计中枢（4.8 的 MIRT、5.5 的 IRT 2PL 任务族权重）。

- **共同病灶**：IRT/MIRT 的 item 参数估计精度由考生数决定。横向对比 seed/deepseek/kimi/glm + 前沿模型，考生数是个位数到十几。这个数量级下，item 难度/判别**不可被数据识别**，估计被先验吞没。

## 5. 基于辩论的 CORE-A v2 修订（改了什么、为什么）

> 不重写全篇，只列**增量 diff**。

- **R1 维度重述**：5 维保留，但从"正交"改为"最小耦合可诊断"；明确 C vs R 边界（§4.4）；强制输出维度相关矩阵；`|r|>0.7` 触发合并。**为什么**：正交性是经验问题，不能口头宣称。

### 6.3 冲突调和（三处最需要协调的张力）

## 7. 收尾：不让步点 / 最大让步 / 三条最关键批评

### 7.1 我坚决不让步的点（1–2 条）
1. **在 `N_models ≈ 5–15` 的现实下，MIRT/IRT 不得作为主聚合或选题/等值化的地基。** 主聚合必须是混合效应模型。这是统计诚实问题，不是口味问题——地基不可识别，上层全部连带失真。我对 4.8 的 MIRT 中枢与 5.5 的 IRT 任务族权重都坚持这一否决。
2. **效率/成本零权重进入 capability 分。** 拒绝 5.5 的"≤15% 乘性折入"。效率只在成功子集上、以 regret + 帕累托**正交**呈现。能力测量的纯度不容 15% 的污染。

### 7.2 我做出的最重要让步
**撤回 task-level 乘法标量聚合（固定指数 `a_1..a_4`）**，并**采纳因果有效性门控**填补我 Round-1 的"环境免费满足"刷分漏洞（这是 4.8 比我强的地方，我用"工具效应归属"实现得更稳健）。同时把效率移出能力分、负分 robustness 改硬门控、judge 硬阈值改分层阈值。

### 7.3 我最关键的三条对他人方案的批评
1. **【命中两者】用个位数到十几个"考生"做 IRT/MIRT 是统计戏剧**：item 判别/难度不可识别，贝叶斯后验被先验吞没，于是"数据驱动权重""Fisher 选题""IRT 等值化"全是循环论证。改用混合效应模型。
2. **【GPT-5.5】内部自相矛盾两处**：§10.3 soft event matching 用 `λ/β` 任意权重 + `sim_*` 相似度软分，正是它自己痛斥的东西；§10.7 把效率折入 `Score_t` 又与 §12.7"主榜不做 cost-adjusted"直接打架。
