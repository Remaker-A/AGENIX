## CP1　聚合机制：分量向量 vs 加法 vs 乘法；是否/如何输出单值

**(a) 最终立场（微调，趋同）：**
- **headline = 维度向量**（每维点估计 + 聚类 bootstrap CI），**不裸报单一总分**。
- **坚决反对乘法聚合**（Opus-4.6 本轮已自行撤回，GPT-5.5 §1.1 也证其会把 `outcome=0` 时的过程部分得分清零）。
- **反对 Pareto-only 纯粹主义**（与 GPT-5.5 §3.2 一致）：5–6 维 × 十几个模型下 Pareto 几乎全员不可比，对"在 seed/deepseek/kimi/glm 里选一个"的用户毫无用处。
- **允许一个可选标量**，但必须套"敏感性外壳"。组件合成在**校准后的尺度（z-score 或 logit）上做加法**，绝不在 outcome×process 上做乘法。

**(b) 逐条回应：**
- 无人实质攻击我 CP1 的立场（我第 1/2 轮就主张向量优先 + 可选标量 + 敏感性）。Opus-4.6 本轮 §4.2 撤回乘法、§4.1 接受"可给标量但带不确定性外壳"，与我合流。GPT-5.5 §3.2 反对 Pareto-only，也与我一致。**三方在此已无真分歧。**

**(c) 可落地裁决规则：**
1. Headline 输出三件套：①维度向量 `{U1…U6}` 各带 95% 聚类 bootstrap CI；②能力–成本 Pareto 前沿；③权重族下的排名稳定区域。
2. 若机构要标量：`Composite(W)=Σ_k W_k · z_k`（`z_k` 为该维度跨模型标准化分；或在 logit 尺度），`W` 三选一来源——**部署画像权重 / 信息量权重 / 等权**，三者必须**全部**列出。
3. 强制敏感性：从 `Dir(α)` 采样 ≥10,000 组权重，报告成对 Kendall's τ；**任意两模型排名翻转概率 > 30% → 判"不可区分"，禁止下"谁更强"结论**（采纳 Opus-4.6 §3.7.2 这条好规则）。
4. **禁止项**：`S=Π 分量^a` 形式；任何把 outcome 与 process 相乘导致"结果失败即过程清零"的写法。短板效应交给"安全/约束 hard-gate"（CP2），不交给乘法。

**(d) 标签：RESOLVED。**

## CP3　统计主干：混合效应模型为主 vs IRT 的角色边界

**(a) 最终立场（重大让步，确认并细化）：**
- **主估计器 = 混合效应模型（GLMM）**，不是 MIRT、不是 per-model 潜在 θ。
- **IRT 的角色收窄到"题目校准"**（难度 `b_i` / 区分度 `a_i`，用于选题、抗饱和、退役饱和项），**且仅在有"合成被试梯"时**才估，**且需通过可识别性自检**。
- **细化我第 2 轮的让步**：第 2 轮我退到"每维单维 2PL/GPCM + per-model θ 带 CI"。**本轮再退一步——连 per-model θ（哪怕单维）都不进 headline**；headline 只有 GLMM 的模型对比 + 可观测维度均值 + CI。这是对两位攻击的彻底回应。

**(b) 逐条回应：**
- GPT-5.5 §2.1 / Opus-4.6 §2.1+§3【高】："9 参数/约 10 被试"欠定、贝叶斯正则=自证（后验被先验吞没 → "数据驱动载荷"退化为作者 `w_i` → 循环论证）、θ 旋转不定、CI 宽到无意义。**全部认账。** 我第 1 轮 §10-Q2 的辩护（先验+主压1–2维+CI）恰恰被两位正确识别为"自认其罪"。
- 对"多 checkpoint/温度凑被试"的反驳（违反局部独立、有效被试仍≈底模数）——**接受**。所以我不靠"灌 checkpoint"救场。
- **我守住的一点（与 GPT-5.5 §2.1 建议一致，Opus-4.6 更保守）**：用**人工合成被试梯**（消融/降级 agent：去重试、截断上下文、去规划步、限工具）可获得**真competence 跨度**的被试群，对**题目参数**估计是合法的——因为这些 agent 的能力差异是真实且单调的，不是同一模型的伪副本。但 IRT 产物**只用于题目侧（选题/ceiling）**，不外推到 per-published-model θ。Opus-4.6 对此更保守（怕合成被试也有问题），故此处保留一个**可经验裁决的窄分歧**：是否允许"合成被试梯上的题目校准 IRT"。

**(c) 可落地裁决规则：**
1. **模型对比主估计器**：
   `logit P(success_{m,t,r}) = β_m + u_template + u_instance (+ u_run)`，
   模型为固定效应、模板/实例（/运行）为随机效应；报告 `β_m` 对比的聚类 bootstrap CI + Holm/BH 多重校正 + 方差分解 + 设计效应 `Deff=1+(m−1)ρ` 定样本量。有界连续分用 beta-GLMM 或 logit 变换。
2. **IRT 准入门（窄用途）**：仅当存在 ≥ `N_synth`（建议 ≥30）个**合成被试**（能力跨度覆盖 headline 模型区间）时，才估题目 `a_i,b_i`；**并须通过可识别性自检**——在模拟数据上做参数恢复，恢复相关 `r(â,a_true)≥0.8` 才采信；否则 IRT 产物仅作内部诊断，不影响选题决策。
3. **禁止**：把 per-published-model 的潜在 θ（任何维度数）放进 headline 或排名。

**(d) 标签：RESOLVED**（主干=GLMM 已三方共识；"合成被试梯题目校准 IRT"作为带自检的可选项，分歧已收窄到可经验裁决，不构成阻塞）。

## CP4　多模态 grounding 路线：合成-符号 GT(headline)+真实生态层 双轨

**(a) 最终立场（确认双轨 + 细化，部分让步）：**
- **双轨成立且必需**，三方已收敛。我**确认**第 2 轮的让步：放弃"几乎全合成"，接受真实生态层。
- **细化分工**：合成-符号 GT 轨负责**"基于 grounding 的推理"headline + 顶端区分探针**（反事实最小对 group-score）+ 反污染每轮重采样；真实-人工 typed GT 轨负责**生态效度**，单独报告。
- **ML 验证器必须先标定可靠性才能进 headline**（与 GPT-5.5 §1.3/§3.3 一致，针对 Opus-4.6 的 GroundingDINO/场景图）。
- **grounding 报为双值向量，不合成单标量**（采纳 Opus-4.6 §6.3 与 GPT-5.5 的"分开报告"）。

**(b) 逐条回应：**
- GPT-5.5 §2.4 / Opus-4.6 §2.3【中】：纯合成生态效度不足，且 matplotlib 风格在训练语料常见、可能**更易**→顶端饱和、降低高区分度。**认账**——尤其"合成更易压低顶端区分度"这一反向风险我第 1 轮没考虑到，是好补充。故合成轨的 headline 角色限定在**反事实最小对/组合关系探针**（这类合成恰恰最难、最抗饱和），而非"读 matplotlib 柱状图"这种易饱和题。
- 我对两位的反向攻击仍成立并保留：**真实资产无法每轮重采样 + gold evidence graph 标注昂贵**（我 §G2、Opus-4.6 §1.2 共同指出 GPT-5.5 的 evidence graph 可扩展性问题）。故真实轨**必须接受"半静态 + 慢轮换 + 标注预算"**，靠 isomorph-gap/canary 监控，而不是假装能廉价再生。
- evidence graph：三方收敛到**"抽取-then-验证 + 报告抽取器 F1"**，且 GT 图只在合成资产上自动派生、用闭式 ID 词表精确匹配（我 §REV-3）。不强制本体序列化（Opus-4.6 §1.2、§2.3）。

**(c) 可落地裁决规则：**
1. `Grounding = {synthetic_groupscore, real_LSVF_score}` **双值**，永不合并为单标量；ceiling 分析分别追踪两轨饱和。
2. **合成轨**：生成器吐符号 GT；headline 用反事实最小对 group-score（image-score ∧ text-score）+ 关键定位 IoU≥0.5；每轮新种子重采样。
3. **真实轨**：小规模人工 typed GT（bbox IoU、CER、表格 TEDS、tIoU、数值 relaxed-acc）；**季度慢轮换 + 明确标注预算**；isomorph-gap 监控记忆。
4. **ML 验证器准入门**：任何 ML 判定器（含 OCR 引擎、检测器、抽取器）须在人工校准集上报告 precision/recall 或 CER，达阈（建议与人工 GT 一致率 ≥0.95）才进 headline；否则仅诊断。
5. 发布**合成-真实相关性 `Pearson(synthetic, real)`** 作为效度证据（采纳 GPT-5.5 §5.4）。

**(d) 标签：RESOLVED（机制层）；但"两轨在任何综合分中的相对权重"留作 IRREDUCIBLE-lite（见 §综合者 与 CP4 尾注）。**

> **CP4 尾注（残留）：** 机制收敛了，但**"谁是被引用的那个 grounding 数字"是价值判断**——若评测目的是"推理上限"则合成轨为主，若目的是"部署感知鲁棒"则真实轨为主。这不是技术分歧，是目的依赖的权重选择，交综合者按目的裁决。

## CP5　因果有效性门控实现：反事实空跑基线 vs 工具效应归属 vs typed verifier

**(a) 最终立场（让步实现，守住原则）：**
- **守住原则**：里程碑计分必须有"by-agent 因果门控"，否则环境免费满足/初始态满足会刷分（这点 GPT-5.5 §1.4 还拿去批 Opus-4.6 缺它，说明原则已被三方认同）。
- **让步实现**：放弃"全局反事实空跑基线"，**采纳工具效应归属（tool-effect attribution / provenance）**——两位独立提出的同一方案，明显更优。

**(b) 逐条回应：**
- GPT-5.5 §2.2 / Opus-4.6 §2.2【中高】：用我自己的 `tf2.config_migration`（schema 在"迁移完第 7 个"才切换）反打——空跑永远到不了第 7 步，反事实世界里 G1/G2 谓词恒假，基线退化。**完全认账。** 我第 1 轮 §10-Q1 已自认隐患，两位正确地把它从"隐患"升级为"地基问题"。
- 采纳 Opus-4.6 §2.2 / GPT-5.5 §4.3 的实现：每个 mock 工具/动作**声明其可写状态变量集**；sandbox 记录每个 state-diff 的 provenance；谓词 `φ_v` 仅在"其依赖的状态变量最后一次被某 agent 动作的因果锥改写"时才计 by-agent。这是**局部、可程序化、与路径无关**的，优于全局 no-op。

**(c) 可落地裁决规则：**
1. **有副作用里程碑**：`by-agent(v) := (φ_v 读取的状态变量在评估时刻最后由 agent 动作集内的动作写入) ∧ ¬(仅由初始快照满足)`。靠 trace 的 `state_diff.provenance` 判定，不重跑。
2. **无副作用/认知型里程碑**（如"是否真的查询了 visa 规则"）：`by-agent(v) := trace 中存在该必要信息获取动作`（布尔存在性），不走状态 provenance。
3. **全局 no-op 仅保留在引擎元测试**（"空 agent 必须 0 分"），不作为 per-milestone 因果判据。
4. typed verifier 负责"谓词真假"，tool-effect attribution 负责"是否 by-agent"，二者正交组合：`a_v = a_v^raw · ∏_{u∈pre(v)} a_u · 1[by-agent(v)]`。

**(d) 标签：RESOLVED。**

## CP6　抗污染 vs 纵向可比：轮换 held-out + 锚点等值化的张力

**(a) 最终立场（让步，确认采纳模板/分布锚定）：**
- **放弃字面 anchor item + IRT-linking**；**采纳"按模板/难度分布锚定 + 新种子同构桥梁集"**（Opus-4.6 §2.4 的修正，正确）。
- 抗污染优先级仍高于跨期可比，但用"模板级桥梁"同时拿到二者。

**(b) 逐条回应（本轮题面点名）：**
- Opus-4.6 §2.4【中高】两个悖论：①字面锚点必须长期不变→最大暴露于污染（我第 1 轮 §10-Q9 自认）；②IRT-linking 依赖不可识别的 item 参数→继承 MIRT 病。**两条都认账。** 我第 2 轮给的"anchor 卫生协议（极小+慢轮换+复用前 isomorph 体检+退役）"方向对，但**根上仍绑在'字面 item + IRT-linking'，没解决悖论②**。所以我本轮**把锚定单位从 item 升到 template/分布**，绕开 IRT-linking。
- 采纳：桥梁集 = 锚**模板**的全新种子同构实例；用**难度分布匹配**（而非 item 参数 linking）做跨版本参照；isomorph-gap 持续监控锚模板是否被记忆，显著则退役换新同构模板。

**(c) 可落地裁决规则：**
1. **无字面锚 item。** 跨版本桥梁 = 一组"锚模板"，每版用全新种子重实例化。
2. **锚模板资格门**：(i) 在合成被试梯上跨种子成功率方差 `< τ_var`（难度稳定，建议 τ_var 使跨种子成功率标准差 <0.05）；(ii) `ContamGap = Acc_orig − Acc_isomorph` 的 bootstrap CI 不显著（未被记忆）；任一不满足 → 不得作锚。
3. **跨版本可比** = 难度分布匹配后的成功率 delta，或用 GLMM 的模板随机效应做桥，**不用 IRT item-parameter linking**。
4. 锚模板轮换节奏慢于主集；触发污染信号即退役。

**(d) 标签：RESOLVED。**

## CP7　headline 指标：pass^k（可靠性）vs pass@k 关系

**(a) 最终立场（部分让步：pass^k 不再是唯一 headline；坚持"可靠性须一等"）：**
- **让步**：撤回"pass^k 作为 agentic/长程唯一 headline"。**headline 主指标 = per-run 成功率（GLMM `β_m` / mean ± CI）**。
- **坚持**：对 agent 而言，**一个可靠性指标必须是一等公民**（per-run 均值会掩盖"每次独立 80%"与"稳定 80%"之别）。但用**properly-estimated 的一致性量**替代朴素 pass^k，并固定/报告温度。
- pass@k = 上限/可达性；长程补**连续量**（期望完成里程碑数）保区分度。

**(b) 逐条回应：**
- GPT-5.5 §2.7【低】：长程 L5 下 pass^k(k=5) 把模型压到≈0、失区分度。**认账**→长程改用连续量（期望里程碑/期望进展）保留区分。
- Opus-4.6 §2.7【中】：pass^k 是 `p^k` 强非线性变换，5 次跑估方差极大（需 `n≫k`）；且温度混杂（temp=0 时 pass^k≈pass@1 平凡，惩罚 temp>0 的创造性）。**认账**→(i) 用**无偏估计量**估 `E[p_i^k]`（先对每实例用足够多次 run 估 `p_i`），要求 `n≥3k`；(ii) 在**注册温度**下报告，且**同时报 temp=0 与部署温度**以暴露混杂。
- **我守住的窄点**：可靠性不能因为"难估"就不报。解决"难估"用更好的估计量与 `n≫k`，而不是删掉可靠性视角。

**(c) 可落地裁决规则：**
1. **headline 成功** = per-instance/per-run 成功率，经 GLMM `β_m ± CI`。
2. **可靠性列**（一等但非唯一 headline）：先用 `n≥3k` 次 run 估每实例 `p_i`，报告一致性 = 无偏 `Ê[p_i^k]`（或"k 次全成率"），**注明 k、n、温度**；temp=0 与部署温度各报一栏。
3. **上限列**：pass@k（注明 k、`n≫k`）。
4. **长程任务**：补"期望完成里程碑数 / 期望进展势能"连续量，防可靠性塌缩失区分。

**(d) 标签：RESOLVED**（估计口径已收敛；"可靠性是否与 per-run 成功率并列 headline"是目的依赖的展示权重，留作 IRREDUCIBLE-lite，见综合者段）。

## CP8　效率 / 成本：是否已完全收敛为"与能力正交、Pareto + 成功子集 regret"

**(a) 最终立场（坚持，且这是我"赢下"的一条；仅让 c\* 的算法）：**
- **完全收敛**：效率/成本与能力**正交**，能力轴**零自由参数**。两位本轮均向我靠拢（GPT-5.5 §3.1 撤回 ≤15% 折入；Opus-4.6 §4.6 撤回 `ρ^β` 折入）。
- **唯一残留我让步**：`c*` 不用"必要性消融重跑"（两位证其组合爆炸/随机/非唯一），改用**强基线分位数 / oracle 最优**。

**(b) 逐条回应：**
- GPT-5.5 §2.3 / Opus-4.6 §2.5【中】：消融重跑代价爆炸、greedy≠最小、随机环境下 `c*` 是分布、oracle 不总存在。**全部认账。** 我第 1 轮的必要性消融在工程上确实不可行。
- 采纳：`c* = min(oracle_optimal, P10(强基线成功轨迹成本))`；随机环境取**重采样中位数 + CI**；无 oracle/强基线的任务**退回 Pareto-only，不强算 regret**（不硬凑）。
- 我守住：成功条件化（失败不享省步红利）、thrash、进展单调性、硬件无关成本单位、cost-adjusted 仅可选视图——三方已无分歧。

**(c) 可落地裁决规则：**
1. 能力分**不含任何效率/成本项**（零自由参数）。
2. 效率仅在 `Success=1` 子集：`Regret = max(0, (c_model − c*)/c*)`；`c* = min(oracle_optimal, P10 强基线成功成本)`；随机环境 `c*` 取重采样中位数并报 CI；无参考 → Pareto-only。
3. 成本主单位 = tokens + 工具调用数（硬件无关）；wall-clock 仅同硬件下报且标注不可跨环境比。
4. 标量 `U = Success − λ·Cost` 仅作**可选采购视图** + `λ` 敏感性，永不进 headline 能力数。
5. 反 gaming：thrash 比率（参数规范化哈希相同且 error 的近似重复占比）+ 进展单调性 `Mono`，替代任何回溯惩罚。

**(d) 标签：RESOLVED。**

## 2. 收束：本轮新让步 / 仍 IRREDUCIBLE 的点（含给综合者的裁决标准）

1. **CP3：** 主估计器从"MIRT/单维 IRT per-model θ"彻底退到 **GLMM**；IRT 仅题目校准 + 须可识别性自检。（比第 2 轮再退一步）

### 2.2 仍判 IRREDUCIBLE 的点（2 条，均为"目的依赖的价值权重"，附裁决标准）
