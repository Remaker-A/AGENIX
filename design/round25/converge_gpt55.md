## CP1 聚合机制：分量向量 vs 加法 vs 乘法；是否/如何输出单值

### (a) 我的最终立场

维度向量为 headline，**不默认输出单一标量**；若需标量，用"标准化后的加性聚合 + Dirichlet 敏感性 + 排名稳定区域"，短板效应交给 **hard-gate** 而非乘法。乘法聚合（Opus-4.6 原案）应被彻底放弃。这一立场较我第 1 轮无变化，但现在三方一致。

### (b) 逐条反驳/认输

- **对 Opus-4.6（已撤回乘法聚合）**：接受其让步。我补一刀确认其撤回是对的——`S_task = S_out^{a1}·S_proc^{a2}·…` 的指数 `a_i` 是"对数空间的固定权重"，且 `0^{a1}` 会在终态失败时把长程过程分清零、并惩罚"结果对但没走预设里程碑"的合法替代路径（Opus-4.8 的 O2 也独立指出这点）。撤回正确，无残留分歧。
- **对 Opus-4.6 的 Pareto-only 倾向**：我坚持反对把 Pareto 支配当主排序。5–6 维、十几个模型下绝大多数模型对 Pareto 不可比，对"横向选型"无用。Opus-4.8 在其 §5.2① 也明确反对纯 Pareto，三方在此已对齐到"向量 + 可选带敏感性标量"。
- **认输项**：无。我第 1 轮的 achievement 聚合 `A_t = μ_F F + μ_P P + μ_G G + μ_R R` 本就是加性，不涉及乘法清零问题；唯一的乘性项是效率调节，已在 CP8 移除。

### (c) 裁决规则

1. 默认 headline = 维度向量 `{U1..U5}`，每维用混合效应边际成功率（logit 尺度）± 95% cluster-bootstrap CI；U6（对抗/安全）单列，**永不并入能力均值**。
2. 可选标量（仅按需）：`Composite(W) = Σ_{k∈U1..U5} W_k · z_k`，其中 `z_k = (D_k − mean_cohort)/sd_cohort`；`W` 取 {部署画像决策论权重 / 等权 / 信息量权重} 之一。**强制**：`W ~ Dirichlet(1)` 采样 ≥10⁴ 次，报告 Kendall's τ 排名稳定性，对每个模型对给出排名翻转概率；翻转概率 >30% 的对标记"统计不可区分"（采纳 Opus-4.6 §3.7.2 这条好规则）。
3. 短板/关键约束由 hard-gate 谓词处理（见 CP2），**不用乘法**；如需短板诊断量，用 `min` 分量与调和平均**仅作诊断**，不进主分。

### (d) 标签：**RESOLVED**

## CP2 安全 / critical 违规处理：硬零不可补偿 vs 软惩罚 / 地板

### (a) 我的最终立场

critical 违规 = **硬零、不可补偿**，并单列 ASR；非 critical 的轻微越界在其里程碑内扣分。采纳 Opus-4.8 的**严重度分级**补丁。这是我第 1/2 轮绝不让步的点，现已成共识。

### (b) 逐条反驳/认输

- **对 Opus-4.6（已认输）**：接受其 R7/§4.7 让步——它放弃了 `(0.5+0.5·max(0,S_rob))` 这个把泄密模型只扣约 13–16% 的软地板，改为 hard-gate。这正是我和 Opus-4.8 都攻击的点（我 §1.2、Opus-4.8 O3），三方现在一致。
- **认输/吸收项**：吸收 Opus-4.8 的"critical vs minor 分级"。我第 1 轮只有 `critical_failure → 0`，没有显式区分 minor 越界；现接受"不是每个约束滑动都该全局清零"。

### (c) 裁决规则

1. 每任务声明 `critical_violation` 集合：secrets 外泄、未授权的不可逆副作用、调用 honeypot、篡改 verifier/GT。命中任一 → 该任务能力分 = 0（不可被其他分补偿）+ `Success = 0` + 记入 ASR 轴并标注攻击强度。该 rollout 在能力模型中**按失败计入**（不丢弃，避免幸存者偏差）。
2. `minor_violation`（如多提交一张工单、轻微预算超支）→ 仅在所属里程碑内扣分，不全局清零。
3. 安全永远不作为可补偿项进入能力标量。

### (d) 标签：**RESOLVED**

## CP3 统计主干：混合效应模型为主 vs IRT 的角色边界

### (a) 我的最终立场

**混合效应模型（GLMM logistic / beta 混合）是统计脊柱**；IRT 降级为"item 难度/区分度校准 + 选题/ceiling"，且 item 校准须跑在**合成被试梯队**上；全维 MIRT 封存到模型样本量足够 + 单载荷锚题时。per-model 潜在 θ 不进 headline。较我第 2 轮，这里我进一步**认输并锁定**：承认 IRT 在真实被试上不可作主干。

### (b) 逐条反驳/认输

- **对 Opus-4.6 §3（"~5–15 考生做 IRT 是统计戏剧"，同时命中我和 4.8）**：**认输（部分）**。它对我 §10.10 用 2PL 估任务族区分度权重的批评成立——item 判别/难度精度由考生数决定，十几个真实模型估不出可信 item 参数。我的修补（合成被试梯队）能救的只是 **item 难度/选题**，**救不了**真实模型的能力标定。因此主干让给混合效应，IRT 不碰真实模型打分。这是我本轮对自己方案的实质让步。
- **对 Opus-4.8（已让步到每维单维 2PL，自称"收敛到 GPT-5.5 的克制用法"）**：接受其 R1 让步。残留的一点小分歧：每维 headline 数应是**混合效应边际估计**，而不是 IRT 的 θ（即便每维 2PL 也带先验吞没风险）；IRT 的 θ 至多作二级视图。Opus-4.8 既已自称收敛到我的克制用法，此点不构成阻塞。
- **对 Opus-4.8 G7（我的混合效应写成高斯线性，设定误用）**：**认输**。有界/二元结果应用 GLMM logistic / beta 混合，在 logit 尺度建模，而非高斯 `y=θ_m+β+u+ε`。采纳。

### (c) 裁决规则

1. 脊柱：`logit P(success_{m,t,r}) = θ_m + s_{family} + u_template + u_instance`（二元）；有界连续子分用 beta 混合回归（logit 尺度）。输出 θ_m 对比 + cluster-bootstrap 95% CI + Holm/BH 校正 + 方差分解（模型/模板/实例/运行）。
2. IRT：仅用**每维 2PL/GPCM** 估 item 难度 `b_i`/区分度 `a_i`，且**拟合在合成被试梯队上**（刻意按能力分级的消融/不同 scaffold/预算 agent，`N_synthetic ≥ 200`）；产物只用于**选题（Fisher 信息）与 ceiling**，**不**用于真实模型打分。
3. 全维 cross-loading MIRT 门槛：仅当 `N_real_models ≥ 25` **且**存在单载荷锚题时启用，且强制报告先验敏感性。per-model 潜在 θ 永不作 headline。

### (d) 标签：**RESOLVED**（边界已明确：混合效应主干、IRT 限 item 校准、MIRT 封存）

## CP4 多模态 grounding 路线：合成-符号 GT（headline）+ 真实生态层 双轨

### (a) 我的最终立场

**双轨成立**：合成-符号 GT 作反污染/可重采样轨，真实媒介 + 人工 typed GT 作生态效度轨，**两值分报、永不合成一个标量**。ML 验证器（GroundingDINO/场景图/OCR 引擎）**必须先标定可靠性**才能进 headline，否则只作诊断。evidence graph **采用闭式词表稳定 ID**，把节点对齐变成精确比对。相较第 2 轮我"真实核心为 headline"的措辞，本轮**让步**：接受合成-符号轨承担反污染 headline，但坚持真实生态层是**强制并报**项、不可降为可选。

### (b) 逐条反驳/认输（含本轮题面点名的两条）

- **对 Opus-4.8 G2（evidence graph 真实媒介 gold 标注不可扩展、与反污染重采样冲突）**：**认输**。20 分钟真实监控视频的逐对象 track/temporal span/bbox 人工标注既贵又无法每轮重采样，与我"template+generator+private params 可规模化"的承诺在真实资产上确实冲突。修正：把"反污染重采样"与"生态效度"**拆到两条轨**——合成轨每 seed 重生成 GT（零标注、抗污染），真实轨一次性人工标注、小规模、静态、用 isomorph-gap 监控污染但不假装抗污染。张力由"两个属性分配到两条不同的轨"化解，而非在同一资产上强求两者。
- **对 Opus-4.8 G3（F1 节点对齐可能回到相似度/judge 循环）**：**认输并采纳其修正**。evidence graph 改为**生成器分配的闭式 ID 词表**（如 `node:order_qty`、`node:video_pick_event`、关系类型枚举），模型引用 ID 而非自创节点 → 节点/边匹配是**精确集合比对**，几何节点（bbox/span）用 IoU/tIoU 与 ID 绑定判定。自由形式输出走"受约束抽取器 → 精确谓词"，并报告抽取器在 held-out 的 F1（抽取器是受测仪器）。这同时消解了 Opus-4.6 §1.2 的"本体顺从度"指控——评分对象是"是否引用了正确 ID 证据"，不是"是否按我的本体序列化"。
- **对 Opus-4.6 的 LSVF 用未标定 ML 验证器**：坚持我的批评。GroundingDINO/场景图当判定器若不报可靠性，就是把"用不可靠仪器测量"从 CLIP 换个模型。它在真实轨可用，但**必须**先标定。

### (c) 裁决规则

1. 架构（三方已同意）：Track-A 合成-符号 GT（每 seed 重采样、抗污染、作 grounded-reasoning 横比 headline）；Track-B 真实/拟真媒介 + 人工 typed GT（小、静态、isomorph-gap 监控、作生态效度）。
2. grounding 报告为两值对象 `{G_synthetic, G_real}`，**禁止合成单一标量**。
3. typed 验证器菜单（仅此类可判定）：bbox/mask `IoU`、mAP；OCR `CER` + 关键字段精确；表格 `TEDS` + 单元格 F1；视频 `tIoU` + 事件顺序；数值 `relaxed accuracy`（`|ŷ−y|/|y| ≤ τ`，财务 τ=0.5%、图表 τ=5%）；组合关系用反事实最小对 `group-score`。
4. evidence graph：闭式 ID 精确匹配 + 几何节点 IoU/tIoU；自由输出走受约束抽取器并报其 F1。
5. ML 验证器准入门槛：仅 Track-B 可用，且 held-out `precision/recall ≥ 0.95`（相关类）或关键字段 `CER ≤ 0.02`，否则只作诊断、不进 headline。
6. **裁决依据（综合者需定）**：跑 pilot，计算各模型 `G_synthetic` 与 `G_real` 的 Spearman ρ。**ρ ≥ 0.8** → `G_synthetic` 作 headline、`G_real` 降为季度审计；**ρ < 0.8** → 两者**并列 headline**（说明它们测的是不同构念，不可互相替代）。

### (d) 标签：**IRREDUCIBLE**（双轨架构、ML 验证器标定、闭式 ID 均已 RESOLVED；唯"真实层是否并列 headline"取决于 pilot 的 ρ，需综合者按上述阈值裁决）

## CP5 因果有效性门控实现：反事实空跑 vs 工具效应归属 vs typed verifier

### (a) 我的最终立场

用**工具效应溯源（tool-effect provenance）**做 `achieved-by-agent` 门控：每个工具声明可写状态变量集合，trace 记录每个 state-diff 的来源，里程碑谓词只在"其依赖状态变量被 agent 工具调用的因果锥改写"时计入。**弃用反事实空跑基线**作主机制（在事件驱动环境退化）。我第 2 轮提的"provenance"与 Opus-4.6 的"工具效应归属"是同一机制，本轮统一命名。

### (b) 逐条反驳/认输

- **对 Opus-4.6 §1.4（指我缺因果门控）**：澄清——我第 2 轮已新增 provenance 归因，与它的"工具效应归属"实为同一思路；本轮合并，无分歧。
- **对 Opus-4.8 的反事实空跑**：坚持反对作主机制。它自己的 `tf2` 里 schema 在"迁移完第 7 个"才切换，空跑永远触发不了，反事实世界退化、`achieved-by-agent` 比较对象不存在（Opus-4.6 §2.2 与我 §2.2 独立指出，Opus-4.8 Q1 自认）。空跑只可作**合成任务上的离线自检**（引擎元测试），不作通用判据。
- **认输项**：无；但采纳"strict 0 地板 + 显式 OR 组"替代我第 1 轮的边容忍系数 `ρ`（见 CP 旁注：Opus-4.8 G5 指 `ρ` 与 Opus-4.6 的 `ε` 同病，我接受删除 `ρ`，软化只来自 OR 结构）。

### (c) 裁决规则

1. `achieved_by_agent(v) = 1` 当且仅当：(a) `φ_v` 引用的所有状态变量都在某个 agent 工具调用的因果锥内被写过（工具声明可写变量集 + trace 记录 diff 来源）；**且** (b) `φ_v` 在初始状态为假（或其为真可证需 agent 写入）；**且** (c) `φ_v` 不是被独立于 agent 的自主环境事件单独满足。
2. 里程碑得分 `a_v = a_v^raw · Π_{u∈pre(v)} a_u · 1[achieved_by_agent(v)]`，**严格 0 门控（无 ε/ρ 地板）**；替代路径用显式 OR-group（组内取 max）。
3. 无副作用的只读必要动作（如"是否查询了 visa 规则"）：用**单一必要事件的布尔存在性检查**，不做相似度匹配。
4. 反事实空跑：降级为合成任务的离线自检（验证 no-op 必得 0）。

### (d) 标签：**RESOLVED**（工具效应溯源胜出，空跑弃用）

## CP6 抗污染 vs 纵向可比：轮换 held-out + 锚点等值化的张力

### (a) 我的最终立场

采纳 Opus-4.6 的**按模板/难度分布锚定 + 新种子同构桥梁集**，**弃用字面 anchor item 与 IRT linking**。纵向等值用"桥梁模板的难度分布匹配"，不用 item 参数 linking（与 CP3 砍掉 IRT 主干一致）。这是我本轮从对手吸收的明确改进。

### (b) 逐条反驳/认输

- **对 Opus-4.8 的字面 anchor + IRT linking**：坚持反对。锚点要长期不变才能做桥，于是最易被训练污染（Opus-4.8 Q9 自认）；且 IRT linking 依赖同样不可识别的 item 参数（CP3），等值化继承 MIRT 的病。Opus-4.8 R3 加了"锚点卫生协议"（极小、慢轮换、复用前 isomorph 体检、退役），方向对，但仍把 IRT linking 留作尺度对齐——这一步应一并去掉。
- **对 Opus-4.6**：采纳其 R8 的"模板/难度分布锚定 + 新种子桥"。这是三方里对该张力最干净的解。
- **认输项**：我第 1/2 轮对"纵向可比"着墨不足，接受以 Opus-4.6 方案补齐。

### (c) 裁决规则

1. 桥梁集 `B` = 跨版本固定的**模板**集合（非实例）；每版本用**新种子**实例化 `B`（不复用字面题）。
2. 纵向等值：按 `B` 的**难度分布**对齐（逐模板平均成功率 ± cluster-bootstrap CI），**不**用 IRT item linking、**不**用字面锚题。
3. 每版本污染体检：`ContamGap = Acc(公开/已知变体) − Acc(私有新种子)`（配对 bootstrap CI）；CI 排除 0 → 退役该模板、换新同构模板。
4. 叠加 canary + impossible-knowledge probe（询问只有私有生成器知道的随机字段）+ 私有轮换。

### (d) 标签：**RESOLVED**（模板/分布锚定胜出，字面锚 + IRT linking 弃用）

## CP7 headline 指标：pass^k（可靠性）vs pass@k

### (a) 我的最终立场

**以混合效应 per-run 成功率 `p̂` 为脊柱，pass@k 与 pass^k 都由 `p̂` 推导（传播 CI）**，在标准化解码下报告；agentic/长程的可靠性 headline 用 pass^k，但始终与 `p̂`、pass@k 并列；长程补连续可靠性量以防 pass^k 塌缩。这调和了 Opus-4.8 的可靠性主张与 Opus-4.6 的方差/温度批评。

### (b) 逐条反驳/认输

- **对 Opus-4.8（pass^k 作 agentic headline）**：采纳方向。pass^k 反映"敢不敢真用"，对 agent 比 pass@k 更相关。
- **对 Opus-4.6（pass^k 高方差、需 n≫k、温度混杂）**：**认其技术批评，并给出更优解**——不要用**经验** pass^k（少次数高方差），而是先用混合效应估 per-run `p̂`，再令 `pass^k = p̂^k`、`pass@k = 1−(1−p̂)^k`，CI 从 `p̂` 的后验/bootstrap 传播。这样**无需 n≫k** 即得两个指标，直接解决方差问题。温度混杂用"标准化解码设定下报告 + 标注解码依赖"处理。
- **认输项**：承认我第 1 轮只做 `n≥5` 均值、未区分 pass@k/pass^k，采纳 Opus-4.8 的区分。

### (c) 裁决规则

1. 脊柱 = 每维混合效应 per-run 成功率 `p̂_{m,t}` ± CI。
2. 派生 `pass@k = 1−(1−p̂)^k`（上限/可达性）与 `pass^k = p̂^k`（可靠性/一致性）：逐任务计算再聚合，CI 由 `p̂` 经 Monte-Carlo 传播（**不**用经验 pass^k，规避高方差）。
3. 在**标准化解码设定**下做横比 + 另报部署设定；显式标注解码依赖。
4. 长程任务额外报连续可靠性量（期望完成里程碑比例 / 每步成功率），防 pass^k 在 L5 塌到 0 丧失区分度。
5. 默认 agentic headline 强调 pass^k，但永远与 `p̂`、pass@k 并列；研究/ceiling 视角强调 pass@k。

### (d) 标签：**RESOLVED**（模型化推导调和方差与可靠性之争）

## CP8 效率/成本：是否已完全收敛为"与能力正交、Pareto + 成功子集 regret"

### (a) 我的最终立场

**已完全收敛**：效率/成本与能力**完全正交**，能力轴零成本自由参数；效率只在成功子集上以 regret + 帕累托呈现。我第 1 轮的 `Score_t` 中 `≤15%` 乘性调节**正式删除**。残留分歧仅在 `c*` 的定义，已可调和。

### (b) 逐条反驳/认输（含题面点名的 §10.7↔§12.7 矛盾）

- **对 Opus-4.8 G1 与 Opus-4.6 §1.3（指我 §10.7 折入效率、与 §12.7 自相矛盾、且 `η_t≤0.15`/`λ`/`C_ref` 是隐藏拍脑袋量）**：**认输**。`η_t≤0.15` 确实是无依据常数，`C_ref` 是未审计超参，把它乘进每个 `Score_t` 再聚合，正是 §12.7 要避免的成本污染能力。**彻底删除**该乘性项，能力分 cost-free。
- **对 Opus-4.8 的必要性消融定义 `c*`**：坚持我和 Opus-4.6 的批评——有状态/随机环境里逐动作消融是组合爆炸 + 路径依赖 + 分母带噪。改用 oracle 最优 / 强基线 P10。
- **认输项**：如上，效率折入是我本轮第二个实质认输。

### (c) 裁决规则

1. 能力轴 = **零成本项**（成本引入的自由参数数 = 0）。
2. 效率（仅成功子集）：`Regret = max(0, (c_model − c*)/c*)`，`Eff = 1/(1+Regret)`。`c*` = 有验证过 oracle solver 的任务取 **oracle 最优成本**；否则取**强基线成功轨迹成本的 P10**；随机环境下 `c*` 取重采样中位数并报 CI。**消融法弃用**。
3. 成本单位：硬件无关（tokens、工具调用数、步数）为主；wall-clock 仅在固定硬件/并发下报告并标注不可跨环境比较。
4. 跨模型视图 = 能力–成本**帕累托前沿**；thrash 比率 + 进展单调性 `Mono` 作诊断；可选 `U = Success − λ·Cost` 附 λ 敏感性，置于**独立视图**，永不进能力数。
